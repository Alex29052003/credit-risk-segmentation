# 🧠 Credit Risk Prediction

## 📌 Описание проекта
В данном проекте решается задача прогнозирования дефолта клиентов банка.  
Цель — определить, какие клиенты с высокой вероятностью не вернут кредит, чтобы снизить финансовые риски.

---

## 📊 Данные
Используется датасет с информацией о клиентах:

- кредитный лимит (LIMIT_BAL)
- возраст (AGE)
- образование (EDUCATION)
- история платежей (PAY_0, PAY_2, …)
- суммы счетов (BILL_AMT)
- платежи (PAY_AMT)

Целевая переменная:
- default payment next month (0 — заплатит, 1 — дефолт)

---

## 🔍 Анализ данных (EDA)
В ходе анализа были выявлены:

- история платежей — ключевой фактор риска  
- клиенты с просрочками чаще уходят в дефолт  
- размер кредитного лимита влияет на вероятность дефолта  
- возраст не оказывает значительного влияния  

---

## 🏗️ Подход

1. Загрузка данных и сохранение в SQL (SQLite)
2. Анализ данных (EDA)
3. Разделение данных на признаки (X) и целевую переменную (y)
4. Деление на обучающую и тестовую выборки
5. Обучение моделей
6. Оценка результатов
7. Настройка порога классификации

---

## 🤖 Модели

### Logistic Regression
Использована как базовая модель для получения начального результата.

### Random Forest
Использована как более сложная модель для выявления нелинейных зависимостей и улучшения качества предсказаний.

---

## 📈 Метрики

Использовались:

- Accuracy — общая точность модели  
- Recall (для класса дефолта) — основная метрика  

Почему важен recall:

Важно не пропустить клиентов, которые могут не вернуть деньги,  
так как это приводит к финансовым потерям для банка.

---

## ⚙️ Настройка порога

По умолчанию используется порог 0.5, однако модель пропускала много рискованных клиентов.

Было принято решение снизить порог до 0.3:

y_pred_custom = (y_prob_rf > 0.3).astype(int)

Результат:

- увеличился recall (модель находит больше рискованных клиентов)  
- модель стала более полезной для бизнеса  

---

## 💡 Вывод

Модель позволяет выявлять клиентов с высоким риском дефолта.

Снижение порога классификации улучшило способность модели находить проблемных клиентов,  
что делает её более применимой в банковской практике.
